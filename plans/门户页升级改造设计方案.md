# 门户页升级改造 & 政策资讯/规章制度管理 设计方案

> 版本：v1.0 | 日期：2026-04-29 | 状态：待确认

---

## 一、项目背景

当前系统入口页（`/` 路由）仅作为各端（政府监管端、物业管理端、商家端、业主微信端、超级管理端）的导航入口，功能单一。现需将其升级为**综合门户页**，使其不仅承载系统入口功能，还能作为政府监管端发布政策资讯、物业管理规章制度的展示窗口，提升系统的信息聚合能力和公共服务价值。

---

## 二、功能范围与边界

### In Scope（范围内）

| 模块 | 功能点 | 优先级 |
|------|--------|--------|
| **门户页** | 整体UI重新设计、Banner轮播、政策资讯展示、规章制度展示、行业动态、系统入口卡片、搜索、底部信息 | P0 |
| **政策资讯管理** | 列表、新增、编辑、删除、详情查看、分类筛选、搜索、置顶、发布/下架 | P0 |
| **规章制度管理** | 列表、新增、编辑、删除、详情查看、分类管理、版本管理、发布/下架 | P0 |
| **门户页访问统计** | 浏览量统计、热门内容排行 | P1 |

### Out of Scope（范围外）

- 业主端/物业管理端的独立门户（仅政府监管端管理门户内容）
- 多语言国际化
- 门户页个性化定制（如用户自定义布局）
- 门户页SEO优化

### 3.2 页面区域详细设计

#### 3.2.1 顶部导航栏
- **左侧**：Logo（🏘️）+ 系统名称 "物业全生命周期管理系统"
- **中间**：一级导航（首页 | 政策资讯 | 规章制度 | 行业动态 | 关于我们）
- **右侧**：全局搜索框 + 登录/用户头像下拉

#### 3.2.2 Banner 轮播区
- 全宽轮播图，支持 3-5 张图片轮播
- 每张图片可配置：标题、副标题、跳转链接（关联到具体政策/制度详情）
- 自动轮播 + 手动切换
- 右下角显示公告滚动条（最新通知滚动播放）

#### 3.2.3 内容展示区（Tab 切换）
- **政策资讯**：展示最新发布的政策法规，卡片式布局，显示标题、摘要、发布日期、分类标签
- **规章制度**：展示物业管理规章制度，类似布局
- **行业动态**：展示物业行业新闻、活动通知等
- **通知公告**：展示系统公告、紧急通知等

每个 Tab 下显示 6 条内容（2行×3列），底部有"更多 >"链接跳转到完整列表。

#### 3.2.4 系统快捷入口区
- 保留原有 5 端卡片设计，但缩小尺寸、更紧凑
- 卡片悬停效果保留
- 增加"快捷入口"标题

#### 3.2.5 侧边栏/底部信息区
- **热门排行**：按浏览量排序的热门政策/制度 TOP 10
- **最新发布**：最近 7 天发布的内容列表
- **政策解读**：精选政策解读文章

#### 3.2.6 底部
- 版权信息、友情链接、联系我们、ICP备案号

### 3.3 交互设计
- 响应式布局，适配 1920×1080 主流分辨率
- 卡片悬停放大效果
- 点击卡片进入详情页（新标签页打开）
- 搜索支持全文检索
- 平滑滚动和过渡动画

## 四、政策资讯管理模块设计

### 4.1 功能概述

政府监管端（`/government`）下的政策资讯管理，用于发布、编辑、管理各类物业相关政策法规、通知公告、指导意见等。

### 4.2 页面结构

```
政府监管端
├── 工作台（Dashboard）
├── 系统管理
├── 政策监管（已有）
│   ├── 政策法规管理（已有 - PolicyManagement.tsx）
│   └── 政策执行督查（已有）
├── 资讯管理（新增）
│   ├── 政策资讯列表（新增）
│   ├── 发布政策资讯（新增）
│   └── 资讯分类管理（新增）
├── 规章制度管理（新增）
│   ├── 规章制度列表（新增）
│   ├── 发布规章制度（新增）
│   └── 制度分类管理（新增）
├── 行业管理
├── 商家监管
├── 投诉处理
├── 数据统计分析
├── 街道物业管理
├── 社区管理
└── 专项管理
```

### 4.3 政策资讯列表页

**路由**：`/government/info/list`

**功能点**：
1. **搜索筛选**：标题搜索、分类筛选、状态筛选（已发布/草稿/已下架）、日期范围筛选
2. **列表展示**：表格形式，列包括：标题、分类、发布状态、置顶状态、浏览量、发布时间、操作
3. **批量操作**：批量删除、批量下架
4. **操作按钮**：查看、编辑、删除、置顶/取消置顶、发布/下架

**列表字段**：
| 字段 | 说明 |
|------|------|
| 标题 | 政策资讯标题，可点击查看详情 |
| 分类 | 如：政策法规、通知公告、指导意见、行业动态 |
| 发布状态 | 已发布 / 草稿 / 已下架 |
| 置顶 | 是否置顶展示 |
| 浏览量 | 门户页浏览次数 |
| 发布时间 | 发布日期 |
| 操作 | 查看、编辑、删除、置顶、发布/下架 |

### 4.4 发布/编辑政策资讯页

**路由**：`/government/info/add` | `/government/info/edit/:id`

**表单字段**：
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 标题 | Input | 是 | 政策资讯标题 |
| 分类 | Select | 是 | 政策法规/通知公告/指导意见/行业动态 |
| 摘要 | TextArea | 是 | 简短摘要，门户页展示 |
| 正文内容 | RichText Editor | 是 | 富文本编辑器（如 react-quill） |
| 封面图片 | Upload | 否 | 列表/门户页展示图 |
| 附件 | Upload | 否 | 支持 PDF/Word 多文件 |
| 标签 | Select (tags) | 否 | 关键词标签 |
| 置顶 | Switch | 否 | 是否置顶到门户页 |
| 发布状态 | Select | 是 | 立即发布 / 存为草稿 |
| 发布时间 | DatePicker | 否 | 定时发布 |

### 4.5 资讯分类管理

**路由**：`/government/info/category`

- 分类列表：ID、分类名称、排序、状态、操作
- 支持新增、编辑、删除分类
- 预置分类：政策法规、通知公告、指导意见、行业动态、政策解读

## 五、物业管理规章制度管理模块设计

### 5.1 功能概述

政府监管端发布的物业管理规章制度，供物业企业、业主查阅。与政策资讯的区别：规章制度更偏向**操作规范、管理标准、作业流程**等，具有更强的指导性和约束力。

### 5.2 规章制度列表页

**路由**：`/government/rule/list`

**功能点**：同政策资讯列表，增加"版本号"字段

**列表字段**：
| 字段 | 说明 |
|------|------|
| 标题 | 规章制度标题 |
| 分类 | 如：安全管理、服务标准、设备管理、财务管理、人员管理 |
| 版本号 | 如 v1.0、v2.0 |
| 发布状态 | 已发布 / 草稿 / 已废止 |
| 生效日期 | 制度生效日期 |
| 浏览量 | 门户页浏览次数 |
| 发布时间 | 发布日期 |
| 操作 | 查看、编辑、删除、废止 |

### 5.3 发布/编辑规章制度页

**路由**：`/government/rule/add` | `/government/rule/edit/:id`

**表单字段**（比政策资讯多版本管理）：
| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 标题 | Input | 是 | 制度标题 |
| 分类 | Select | 是 | 安全管理/服务标准/设备管理/财务管理/人员管理/综合管理 |
| 版本号 | Input | 是 | 自动生成 v1.0，修订时递增 |
| 摘要 | TextArea | 是 | 简短摘要 |
| 正文内容 | RichText Editor | 是 | 富文本编辑器 |
| 附件 | Upload | 否 | 支持 PDF/Word |
| 生效日期 | DatePicker | 是 | 制度生效日期 |
| 发布状态 | Select | 是 | 立即发布 / 存为草稿 |
| 标签 | Select (tags) | 否 | 关键词标签 |

### 5.4 制度分类管理

**路由**：`/government/rule/category`

- 预置分类：安全管理、服务标准、设备管理、财务管理、人员管理、综合管理
- 支持自定义新增、编辑、删除、排序

### 5.5 版本管理

- 每次修订创建新版本，旧版本保留在历史记录中
- 版本号规则：v{major}.{minor}
- 版本历史列表：版本号、修订日期、修订人、修订说明

## 六、数据模型设计

### 6.1 政策资讯表（PolicyInfo）

```typescript
interface PolicyInfo {
  id: number;
  title: string;                    // 标题
  category: string;                 // 分类编码
  categoryName: string;             // 分类名称
  summary: string;                  // 摘要
  content: string;                  // 正文（HTML）
  coverImage?: string;              // 封面图URL
  attachments?: string[];           // 附件列表
  tags?: string[];                  // 标签
  isTop: boolean;                   // 是否置顶
  status: 'draft' | 'published' | 'archived';  // 状态
  viewCount: number;                // 浏览量
  publishTime?: string;             // 发布时间
  createBy: string;                 // 创建人
  createTime: string;               // 创建时间
  updateTime: string;               // 更新时间
}
```

### 6.2 规章制度表（ManagementRule）

```typescript
interface ManagementRule {
  id: number;
  title: string;                    // 标题
  category: string;                 // 分类编码
  categoryName: string;             // 分类名称
  version: string;                  // 版本号
  summary: string;                  // 摘要
  content: string;                  // 正文（HTML）
  attachments?: string[];           // 附件列表
  tags?: string[];                  // 标签
  status: 'draft' | 'published' | 'deprecated';  // 状态
  effectiveDate: string;            // 生效日期
  viewCount: number;                // 浏览量
  publishTime?: string;             // 发布时间
  createBy: string;                 // 创建人
  createTime: string;               // 创建时间
  updateTime: string;               // 更新时间
}

interface RuleVersion {
  id: number;
  ruleId: number;                   // 关联制度ID
  version: string;                  // 版本号
  changeLog: string;                // 修订说明
  createdBy: string;                // 修订人
  createTime: string;               // 修订时间
}
```

### 6.3 分类表（Category）

```typescript
interface Category {
  id: number;
  type: 'policy' | 'rule';         // 分类类型：政策资讯/规章制度
  name: string;                     // 分类名称
  code: string;                     // 分类编码
  sortOrder: number;                // 排序
  status: 0 | 1;                   // 状态：0禁用 1启用
}
```

### 6.4 Banner 配置表（PortalBanner）

```typescript
interface PortalBanner {
  id: number;
```
