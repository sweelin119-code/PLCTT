## 九、技术实现建议

### 9.1 代码组织规范
```
src/pages/{端名}/{功能名}/
  ├── index.tsx          # 主页面组件
  ├── components/        # 子组件
  ├── hooks.ts           # 自定义 hooks
  └── constants.ts       # 常量定义
```

### 9.2 状态管理策略
- **跨页面状态：** React Context（AuthContext / CommunityContext）
- **页面内状态：** useState + useReducer
- **服务端状态：** 直接调用 services 层（当前为 mock 数据，后续替换为真实 API）

### 9.3 Mock 数据策略
- 当前所有 services 返回 mock 数据
- 建议在 `src/services/mockData.ts` 中统一管理 mock 数据
- 后续替换为真实 API 时只需修改 services 层，页面无需改动

### 9.4 建议的开发顺序（按批次执行）
1. **第1批** → 业主端H5 4个页面 + 物业端装修管理 + 建议管理
2. **第3批** → 业委会端全功能（与第1批并行开发）
3. **第2批** → 物业端日常管理 + 合同管理 + 房屋/车位费用管理
4. **第4批** → 物业端专业岗位管理（保安/保洁/维保/绿化等）
5. **第5批** → 商家端全功能
6. **第6批** → 业主端H5 生活服务/社交/快递
7. **第7批** → 政府端行业管理 + 数据统计
8. **第8-9批** → 超管端辅助管理 + 政府端专项管理
9. **第10-11批** → 数据中台 + IoT + 工作流引擎
