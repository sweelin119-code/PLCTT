# 收费管理模块设计方案

> 版本：v1.0 | 日期：2026-04-29 | 状态：待确认

---

## 一、项目背景

当前物业管理端已完成工作台 Dashboard、企业入驻、商家管理、投诉处理、维修工单、业主端配置管理等模块的开发。**收费管理**作为物业公司的核心业务模块，直接关系到物业公司的收入命脉，是系统必须优先完善的关键功能。

根据思维导图规划，收费管理涵盖：物业费管理、停车费管理、维修资金管理、财务报表等。但实际业务中，物业收费远不止这三项，还需包含**公摊费、代收代缴费、增值服务费、押金/保证金**等多种费用类型。

本方案参考了市场上主流物业管理系统（如：**千丁、彩生活、万科睿服务、海纳云**等）的收费模块设计思路，结合本系统现有架构，制定一套**灵活、可扩展、覆盖全场景**的收费管理方案。

---

## 二、功能范围与边界

### In Scope（范围内）

| 模块 | 功能点 | 优先级 |
|------|--------|--------|
| **费用项目管理** | 费用项定义（物业费/停车费/公摊费/代收费/押金等）、计费规则配置、单价/周期设置 | P0 |
| **收费标准管理** | 按房屋类型/面积/楼层/户型等多维度定价、阶梯定价、时段定价 | P0 |
| **账单生成** | 周期性自动生成（月/季/年）、手动生成、批量生成、临时账单 | P0 |
| **账单管理** | 账单列表、账单详情、账单调整（减免/退款/冲抵）、账单作废 | P0 |
| **在线缴费** | 微信/支付宝扫码支付、线下缴费登记、聚合支付对接 | P0 |
| **欠费催缴** | 自动催缴（短信/公众号/APP推送）、催缴记录、催缴模板管理 | P1 |
| **收费报表** | 收费率统计、欠费明细、收费趋势、按楼栋/业态汇总 | P1 |
| **业主端缴费** | 业主微信H5端账单查询、在线缴费、缴费历史、电子发票 | P1 |

### Out of Scope（范围外）

- 银行自动代扣（需银行接口对接）
- 电子发票对接税务系统（需税务资质）
- 维修资金使用申请与分摊流程（后续单独模块实现）
- 预算管理与实际收支对比（后续财务模块实现）
- 第三方支付平台商户号申请与配置

## 三、业务模型设计

### 3.1 核心数据模型

```
┌─────────────────────────────────────────────────────────────────┐
│                         FeeItem（费用项目）                        │
│  id, name, code, category, unit, billingCycle, taxRate, status  │
└──────────────────────────┬──────────────────────────────────────┘
                           │ 1
                           │ N
┌──────────────────────────┴──────────────────────────────────────┐
│                    ChargeRule（收费标准规则）                       │
│  id, feeItemId, houseType, pricingMode, formula, price,         │
│  effectiveDate, expireDate, status                               │
└──────────────────────────┬──────────────────────────────────────┘
                           │ 1
                           │ N
┌──────────────────────────┴──────────────────────────────────────┐
│                     Bill（账单）                                   │
│  id, billNo, houseId, ownerId, feeItemId, period(账期),         │
│  amount, paidAmount, discountAmount, status, dueDate,           │
│  createTime, paidTime                                            │
└──────────────────────────┬──────────────────────────────────────┘
                           │ 1
                           │ N
┌──────────────────────────┴──────────────────────────────────────┐
│                   PaymentRecord（缴费记录）                        │
│  id, billId, paymentNo, payMethod, payAmount, payTime,          │
│  operatorId, remark, tradeNo(第三方交易号)                        │
└─────────────────────────────────────────────────────────────────┘
```

### 3.2 费用项目分类（FeeItem Category）

| 大类 | 费用项 | 计费方式 | 周期 |
|------|--------|----------|------|
| **物业费** | 物业管理服务费 | 按面积×单价 | 月/季/年 |
| **停车费** | 车位管理费、车位租金、临停费 | 固定/按次/按时 | 月/年/次 |
| **公摊费** | 公摊水费、公摊电费、公摊燃气费 | 按户/按面积分摊 | 月 |
| **代收代缴** | 代收水费、代收电费、代收燃气费 | 按表读数 | 月 |
| **专项费用** | 维修资金、装修保证金、装修垃圾清运费 | 固定/按面积 | 一次性 |
| **增值服务费** | 家政服务、代收快递、特约维修 | 按次/按项目 | 次 |
| **其他** | 滞纳金、门禁卡工本费、押金 | 固定 | 一次性 |

### 3.3 计费规则模式

| 模式 | 说明 | 适用场景 |
|------|------|----------|
| **固定单价** | 每户固定金额 | 车位管理费、垃圾清运费 |
| **按面积计费** | 房屋面积 × 单价 | 物业费（最常见） |
| **按户计费** | 每户固定金额 | 公摊费均摊部分 |
| **阶梯计费** | 用量阶梯×不同单价 | 水电气代收 |
| **按次计费** | 每次固定金额 | 临停费、家政服务 |
| **按时计费** | 时长×单价 | 临时停车 |
| **公式计费** | 自定义公式计算 | 复杂计费场景 |

## 四、业务流程

### 4.1 收费管理主流程

```
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│ 费用项目  │ → │ 收费标准  │ → │ 账单生成  │ → │ 业主缴费  │ → │ 对账核销  │
│ 配置      │    │ 设置      │    │          │    │          │    │          │
└──────────┘    └──────────┘    └──────────┘    └──────────┘    └──────────┘
                                                                    │
                                                                    ▼
                                                              ┌──────────┐
                                                              │ 报表统计  │
                                                              └──────────┘
```

### 4.2 账单生成流程

```
┌─────────────────────────────────────────────────────────────────┐
│  定时任务 / 手动触发                                               │
└──────────────────────────┬──────────────────────────────────────┘
                           ▼
┌─────────────────────────────────────────────────────────────────┐
│  查询所有生效的收费标准规则                                         │
│  → 按房屋/业主维度分组                                            │
└──────────────────────────┬──────────────────────────────────────┘
                           ▼
┌─────────────────────────────────────────────────────────────────┐
│  检查该账期是否已生成账单（避免重复）                                 │
└──────────────────────────┬──────────────────────────────────────┘
                           ▼
┌─────────────────────────────────────────────────────────────────┐
│  根据计费规则计算每户应缴金额                                       │
│  → 物业费：面积 × 单价                                             │
│  → 公摊费：总用量 ÷ 总户数 × 单价                                   │
│  → 停车费：固定金额                                                │
└──────────────────────────┬──────────────────────────────────────┘
                           ▼
┌─────────────────────────────────────────────────────────────────┐
│  批量生成账单 → 状态为"待缴费"                                      │
│  → 推送缴费通知给业主                                              │
└─────────────────────────────────────────────────────────────────┘
```

### 4.3 缴费流程

```
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│ 业主查看  │ → │ 选择账单  │ → │ 选择支付  │ → │ 支付完成  │
│ 账单列表  │    │ 确认金额  │    │ 方式      │    │          │
└──────────┘    └──────────┘    └──────────┘    └──────────┘
                                                              │
                    ┌─────────────────────────────────────────┘
                    ▼
┌─────────────────────────────────────────────────────────────────┐
│  支付方式分支：                                                    │
│  ├─ 微信支付 → 生成支付二维码 → 业主扫码 → 支付回调 → 更新账单状态  │
│  ├─ 支付宝支付 → 同上                                              │
│  ├─ 线下转账 → 物业后台手动确认到账 → 登记缴费记录                   │
│  └─ 现金支付 → 物业后台手动登记 → 打印收据                          │
└─────────────────────────────────────────────────────────────────┘
```

### 4.4 催缴流程

```
┌─────────────────────────────────────────────────────────────────┐
│  定时任务：每日检查到期未缴费账单                                     │
│  → 到期前3天：首次提醒（温和）                                      │
│  → 到期当天：缴费提醒                                                │
│  → 逾期7天：催缴通知（加急）                                        │
│  → 逾期30天：律师函/上门催缴（系统生成催缴记录）                      │
└─────────────────────────────────────────────────────────────────┘
```

## 六、数据库表结构建议

### 6.1 费用项目表（fee_item）

```sql
CREATE TABLE fee_item (
  id            INT PRIMARY KEY AUTO_INCREMENT,
  name          VARCHAR(100) NOT NULL COMMENT '费用项名称',
  code          VARCHAR(50) NOT NULL COMMENT '费用项编码',
  category      VARCHAR(50) NOT NULL COMMENT '费用类别: property_fee/parking/public/agency/deposit/value_added/other',
  unit          VARCHAR(20) COMMENT '计费单位: 元/㎡/月, 元/月, 元/吨, 元/次',
  billing_cycle VARCHAR(20) NOT NULL DEFAULT 'month' COMMENT '计费周期: month/quarter/year/one_time',
  tax_rate      DECIMAL(5,2) DEFAULT 0.00 COMMENT '税率(%)',
  sort_order    INT DEFAULT 0 COMMENT '排序号',
  enabled       TINYINT(1) DEFAULT 1 COMMENT '是否启用',
  remark        VARCHAR(500) COMMENT '备注',
  create_time   DATETIME DEFAULT CURRENT_TIMESTAMP,
  update_time   DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  UNIQUE KEY uk_code (code)
) COMMENT '费用项目表';
```

### 6.2 收费标准表（charge_rule）

```sql
CREATE TABLE charge_rule (
  id              INT PRIMARY KEY AUTO_INCREMENT,
  fee_item_id     INT NOT NULL COMMENT '关联费用项目ID',
  house_type      VARCHAR(50) NOT NULL COMMENT '适用房屋类型: residence/shop/office/warehouse',
  pricing_mode    VARCHAR(50) NOT NULL COMMENT '计费模式: fixed/area/ household/step/ per_time/ per_hour/formula',
  price           DECIMAL(12,4) NOT NULL COMMENT '单价',
  formula         VARCHAR(500) COMMENT '自定义公式(公式模式时使用)',
  effective_date  DATE NOT NULL COMMENT '生效日期',
  expire_date     DATE COMMENT '失效日期(空=长期有效)',
  enabled         TINYINT(1) DEFAULT 1,
  create_time     DATETIME DEFAULT CURRENT_TIMESTAMP,
  update_time     DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  KEY idx_fee_item (fee_item_id),
  KEY idx_house_type (house_type)
) COMMENT '收费标准表';
```

### 6.3 账单表（bill）

```sql
CREATE TABLE bill (
  id              INT PRIMARY KEY AUTO_INCREMENT,
  bill_no         VARCHAR(50) NOT NULL COMMENT '账单编号',
  house_id        INT NOT NULL COMMENT '房屋ID',
  owner_id        INT COMMENT '业主ID',
  fee_item_id     INT NOT NULL COMMENT '费用项目ID',
  period_year     INT NOT NULL COMMENT '账期年',
  period_month    INT COMMENT '账期月',
  amount          DECIMAL(12,2) NOT NULL DEFAULT 0.00 COMMENT '应缴金额',
  paid_amount     DECIMAL(12,2) NOT NULL DEFAULT 0.00 COMMENT '已缴金额',
  discount_amount DECIMAL(12,2) DEFAULT 0.00 COMMENT '减免金额',
  late_fee        DECIMAL(12,2) DEFAULT 0.00 COMMENT '滞纳金',
  status          VARCHAR(20) NOT NULL DEFAULT 'pending' COMMENT '状态: pending/paid/overdue/ cancelled/partially_paid',
  due_date        DATE COMMENT '缴费截止日期',
  paid_time       DATETIME COMMENT '缴清时间',
  remark          VARCHAR(500) COMMENT '备注',
  create_time     DATETIME DEFAULT CURRENT_TIMESTAMP,
  update_time     DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  UNIQUE KEY uk_bill_no (bill_no),
  KEY idx_house (house_id),
  KEY idx_owner (owner_id),
  KEY idx_period (period_year, period_month),
  KEY idx_status (status)
) COMMENT '账单表';
```

### 6.4 缴费记录表（payment_record）

```sql
CREATE TABLE payment_record (
```
