## 一、业主端H5 - 我的账单

### 1.1 需求概述

为业主提供统一的费用账单入口，聚合展示所有类型的待缴费用（物业费、水电费、停车费），支持在线支付、历史查询、电子发票下载。

### 1.2 页面结构

```
我的账单
├── 账单总览（头部）
│   ├── 总待缴金额（醒目展示）
│   ├── 本月应缴金额
│   └── 欠费笔数提醒（如有欠费红色高亮）
├── 账单分类Tab
│   ├── 全部账单
│   ├── 物业费
│   ├── 水电费
│   └── 停车费
├── 账单列表（按时间倒序）
│   ├── 账单卡片
│   │   ├── 费用类型标签（物业费/水费/电费/停车费）
│   │   ├── 费用周期（如：2026年5月）
│   │   ├── 金额
│   │   ├── 状态标签（待缴费/已缴清/已逾期）
│   │   └── 缴费截止日期
│   │   └── 操作按钮（去缴费/查看详情）
│   └── 加载更多（分页）
├── 缴费记录（历史）
│   └── 已缴费列表（含支付方式、支付时间、发票状态）
└── 电子发票
    ├── 已开发票列表
    └── 申请开票（选择已缴费账单）
```

### 1.3 交互流程

```
业主进入「我的账单」
    │
    ├── 加载账单数据（调用 feeService.getBills）
    │   ├── 成功 → 展示账单总览 + 列表
    │   └── 失败 → 展示空状态 + 重试按钮
    │
    ├── 点击「去缴费」
    │   ├── 进入缴费确认页
    │   ├── 展示账单明细 + 金额
    │   ├── 选择支付方式（微信支付/支付宝）
    │   └── 确认支付 → 调起支付 SDK
    │       ├── 成功 → 跳转支付成功页
    │       └── 失败 → 提示重试
    │
    ├── 切换分类Tab
    │   └── 过滤展示对应类型账单
    │
    └── 点击「查看详情」
        └── 展示账单完整明细（费用项、计算方式、周期等）
```

### 1.4 数据字段

```typescript
interface BillItem {
  id: string;
  houseId: string;
  billType: 'property' | 'water' | 'electric' | 'parking';
  billTypeName: string;
  period: string;           // 费用周期，如 "2026-05"
  amount: number;           // 应缴金额
  paidAmount: number;       // 已缴金额
  status: 'pending' | 'paid' | 'overdue';
  dueDate: string;          // 缴费截止日期
  paidAt?: string;          // 缴费时间
  paymentMethod?: string;   // 支付方式
  invoiceStatus: 'none' | 'applied' | 'issued';
  details: BillDetail[];    // 费用明细
  createdAt: string;
}

interface BillDetail {
  itemName: string;         // 费用项名称，如"物业管理费"
  unitPrice: number;        // 单价
  quantity: number;         // 数量（如面积）
  amount: number;           // 金额
}
```

### 1.5 校验规则

| 规则 | 说明 |
|------|------|
| 金额校验 | 支付金额必须等于应缴金额，不允许部分支付 |
| 逾期标识 | 超过截止日期未缴费自动标记为逾期，逾期账单加收滞纳金（按日计算） |
| 重复支付 | 同一账单不可重复支付，支付前检查状态 |
| 支付超时 | 支付会话有效期 30 分钟，超时自动取消 |

### 1.6 非功能需求

| 项目 | 要求 |
|------|------|
| 响应速度 | 账单列表加载 ≤ 2秒 |
| 支付安全 | 接入微信/支付宝 SDK，敏感信息不经过前端 |
| 数据一致性 | 支付成功后实时更新账单状态 |
| 离线能力 | 已加载的账单数据支持本地缓存查看 |

## 二、业主端H5 - 停车缴费

### 2.1 需求概述

为业主提供车位管理、停车费缴纳、车辆出入记录查看、车位共享功能。

### 2.2 页面结构

```
停车缴费
├── 我的车位（头部）
│   ├── 绑定车位列表（车位号、位置）
│   ├── 车位状态（空闲/已租/到期）
│   └── 快捷操作（缴费/续租）
├── 停车费账单
│   ├── 待缴停车费列表
│   │   ├── 费用周期
│   │   ├── 金额（车位管理费/租赁费）
│   │   └── 去缴费按钮
│   └── 缴费历史
├── 车辆出入记录
│   ├── 记录列表（时间、车牌、出入口、类型）
│   └── 按日期筛选
├── 车位租赁
│   ├── 可租车位列表
│   ├── 租赁价格展示
│   └── 在线租赁申请
└── 车位共享
    ├── 我的共享设置（时段、价格）
    ├── 共享收益记录
    └── 共享车位管理
```

### 2.3 交互流程

```
业主进入「停车缴费」
    │
    ├── 展示已绑定车位信息
    ├── 加载停车费账单（调用 feeService.getParkingBills）
    │
    ├── 点击「去缴费」
    │   └── 同「我的账单」支付流程
    │
    ├── 点击「车辆出入记录」
    │   └── 展示车辆出入时间线
    │
    ├── 点击「车位租赁」
    │   ├── 选择可租车位
    │   ├── 选择租赁时长
    │   ├── 确认租赁 → 生成租赁订单 → 支付
    │   └── 租赁成功 → 车位绑定到名下
    │
    └── 点击「车位共享」
        ├── 设置共享时段和价格
        ├── 发布共享
        └── 查看共享收益
```

### 2.4 数据字段

```typescript
interface ParkingInfo {
  id: string;
  houseId: string;
  parkingNo: string;        // 车位号
  location: string;         // 位置描述
  type: 'owned' | 'rented'; // 自有/租赁
  status: 'idle' | 'occupied' | 'expired';
  rentExpireDate?: string;  // 租赁到期日
  monthlyFee: number;       // 月管理费
}

interface ParkingBill {
  id: string;
  parkingId: string;
  billType: 'management' | 'rent'; // 管理费/租赁费
  period: string;
  amount: number;
  status: 'pending' | 'paid' | 'overdue';
  dueDate: string;
}

interface VehicleRecord {
  id: string;
  plateNo: string;
  entryTime: string;
  exitTime?: string;
  entrance: string;         // 入口名称
  type: 'owner' | 'visitor' | 'temp';
  fee?: number;             // 临时停车费
}
```

### 2.5 校验规则

| 规则 | 说明 |
|------|------|
| 车位绑定 | 一户最多绑定 2 个车位 |
| 租赁冲突 | 同一车位不可同时被多人租赁 |
| 共享限制 | 仅自有车位可共享，租赁车位不可共享 |
| 费用计算 | 临时停车按小时计费，月租按月计费 |

## 三、业主端H5 - 智能门禁

### 3.1 需求概述

为业主提供手机开门功能（蓝牙/二维码/远程），支持访客授权和开门记录查询。

### 3.2 页面结构

```
智能门禁
├── 快捷开门（首页焦点）
│   ├── 默认门禁（小区大门）
│   ├── 单元门禁
│   └── 切换门禁下拉
├── 开门方式
│   ├── 蓝牙开门（靠近自动感应）
│   ├── 二维码开门（生成动态二维码）
│   └── 远程开门（点击按钮）
├── 访客授权
│   ├── 生成临时密码
│   │   ├── 有效期设置（1小时/4小时/1天/自定义）
│   │   └── 分享密码（复制/微信分享）
│   ├── 远程开门（为访客开门）
│   └── 访客记录
├── 开门记录
│   ├── 记录列表（时间、门禁名称、开门方式）
│   └── 按日期筛选
└── 门禁设置
    ├── 默认门禁设置
    └── 常用门禁管理
```

### 3.3 交互流程

```
业主进入「智能门禁」
    │
    ├── 默认展示小区大门门禁
    │
    ├── 蓝牙开门
    │   ├── 检测蓝牙连接状态
    │   ├── 已连接 → 点击开门 → 发送指令 → 开门成功
    │   └── 未连接 → 提示打开蓝牙
    │
    ├── 二维码开门
    │   ├── 生成动态二维码（30秒刷新）
    │   ├── 在门禁扫码器上扫描
    │   └── 验证通过 → 开门
    │
    ├── 远程开门
    │   └── 点击开门按钮 → 发送网络请求 → 门禁开启
    │
    ├── 访客授权
    │   ├── 生成临时密码 → 设置有效期 → 分享给访客
    │   └── 访客在门禁上输入密码 → 验证通过 → 开门
    │
    └── 开门记录
        └── 展示历史开门记录列表
```

### 3.4 数据字段

```typescript
interface DoorDevice {
  id: string;
  name: string;             // 门禁名称，如"小区东门"
  type: 'main' | 'unit' | 'underground';
  bluetoothMac?: string;    // 蓝牙 MAC 地址
  isOnline: boolean;
  isDefault: boolean;
}

interface AccessRecord {
  id: string;
  doorName: string;
  method: 'bluetooth' | 'qrcode' | 'remote' | 'password' | 'face';
  status: 'success' | 'failed';
  time: string;
  visitorName?: string;     // 访客姓名
}

interface TempPassword {
  id: string;
  password: string;
  doorId: string;
  doorName: string;
  validFrom: string;
  validTo: string;
  status: 'active' | 'used' | 'expired';
  visitorName?: string;
  visitorPhone?: string;
}
```

### 3.5 校验规则

| 规则 | 说明 |
|------|------|
| 二维码刷新 | 动态二维码每 30 秒自动刷新，防止截图复用 |
| 临时密码时效 | 过期后自动失效，不可使用 |
| 远程开门频率 | 同一门禁 10 秒内不可重复远程开门 |
| 访客授权上限 | 同时生效的临时密码不超过 5 个 |
| 开门权限 | 仅绑定房屋的业主/住户可开门，租客权限由业主设置 |

### 3.6 非功能需求

| 项目 | 要求 |
|------|------|
| 响应速度 | 开门指令响应 ≤ 2秒 |
| 安全性 | 动态二维码加密生成，防篡改 |
| 离线可用 | 蓝牙开门支持离线模式（本地缓存密钥） |

## 四、业主端H5 - 装修申请

### 4.1 需求概述

业主在线提交装修备案申请，上传相关资料，缴纳装修保证金，查看审批和巡查进度。

### 4.2 页面结构

```
装修申请
├── 我的装修申请（列表）
│   ├── 申请卡片
│   │   ├── 房屋地址
│   │   ├── 申请时间
│   │   ├── 状态标签（待审核/审核通过/施工中/待验收/已完工/已驳回）
│   │   └── 操作按钮（查看详情/继续施工）
│   └── 新建申请按钮（FAB）
├── 新建装修申请（表单页）
│   ├── 选择房屋（如有多个房产）
│   ├── 装修类型（局部装修/全面装修/拆改）
│   ├── 预计施工时间（开始日期 ~ 结束日期）
│   ├── 施工单位信息（公司名称/联系人/电话）
│   ├── 施工人员信息（姓名/身份证号/电话）→ 支持添加多人
│   ├── 上传资料
│   │   ├── 装修方案（图片/PDF）
│   │   ├── 施工图纸（图片/PDF）
│   │   ├── 施工单位资质（图片）
│   │   └── 身份证复印件（图片）
│   ├── 装修承诺书勾选
│   └── 提交申请按钮
├── 申请详情页
│   ├── 申请信息（只读）
│   ├── 审批进度时间线
│   │   ├── 已提交
│   │   ├── 物业审核（通过/驳回及原因）
│   │   ├── 保证金缴纳（金额 + 去缴纳按钮）
│   │   ├── 施工中（剩余天数）
│   │   ├── 物业巡查记录（巡查时间/结果）
│   │   └── 完工验收（通过/需整改）
│   └── 保证金状态（已缴纳/已退还/金额）
└── 缴纳保证金
    ├── 保证金金额展示
    ├── 支付方式选择
    └── 确认支付
```

### 4.3 交互流程

```
业主进入「装修申请」
    │
    ├── 展示已有申请列表
    │
    ├── 点击「新建申请」
    │   ├── 填写表单（房屋、类型、时间、施工方、人员）
    │   ├── 上传资料（图片/PDF）
    │   ├── 勾选装修承诺书
    │   └── 提交 → 状态变为「待审核」
    │
    ├── 物业审核通过后
    │   ├── 收到通知
    │   ├── 进入详情页 → 点击「缴纳保证金」
    │   └── 支付成功 → 状态变为「施工中」
    │
    ├── 施工期间
    │   ├── 查看物业巡查记录
    │   └── 收到巡查异常通知（如有）
    │
    └── 完工后
        ├── 物业验收通过 → 保证金退还
        └── 物业验收需整改 → 整改后再次验收
```

### 4.4 数据字段

```typescript
interface DecorationApplication {
  id: string;
  houseId: string;
  houseAddress: string;
  decorationType: 'partial' | 'full' | 'demolition';
  startDate: string;
  endDate: string;
  companyName?: string;
  companyContact?: string;
  companyPhone?: string;
  workers: DecorationWorker[];
  attachments: string[];        // 上传文件 URL 列表
  status: DecorationStatus;
  depositAmount: number;        // 保证金金额
  depositPaid: boolean;
  depositRefunded: boolean;
  createdAt: string;
  updatedAt: string;

interface DecorationWorker {
  name: string;
  idCard: string;
  phone: string;
  photo?: string;               // 人员照片
}

type DecorationStatus = 
  | 'pending_review'      // 待审核
  | 'approved'            // 审核通过（待缴保证金）
  | 'deposit_paid'        // 已缴保证金（待施工）
  | 'in_progress'         // 施工中
  | 'pending_inspection'  // 待验收
  | 'completed'           // 已完工
  | 'rejected'            // 已驳回
  | 'cancelled';          // 已取消

interface DecorationInspection {
  id: string;
  applicationId: string;
  inspector: string;
  inspectTime: string;
  result: 'pass' | 'rectify';   // 通过/需整改
  rectificationItems?: string[]; // 整改项
  remark?: string;
  photos?: string[];
}
```

### 4.5 校验规则

| 规则 | 说明 |
|------|------|
| 重复申请 | 同一房屋同一时间只能有一个进行中的装修申请 |
| 施工期限 | 装修施工期最长 90 天，到期自动提醒 |
| 保证金标准 | 局部装修 2000 元，全面装修 5000 元，拆改 10000 元 |
| 人员上限 | 同一申请最多添加 10 名施工人员 |
| 资料必传 | 装修方案、施工图纸为必传项 |

### 5.3 交互流程


### 10.4 校验规则

| 规则 | 说明 |
