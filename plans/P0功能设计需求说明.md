## 一、业主端H5 - 我的账单

### 1.1 需求概述

为业主提供统一的费用账单入口，按「房屋费用」和「停车费用」两大类别聚合展示所有待缴费用。房屋费用包含物业费、公摊水费、公摊电费等与房屋相关的费用；停车费用包含停车费、车位管理费等与车位相关的费用。支持在线支付、历史查询、电子发票下载。

### 1.2 页面结构

```
我的账单
├── 账单总览（头部）
│   ├── 总待缴金额（醒目展示）
│   ├── 本月应缴金额
│   └── 欠费笔数提醒（如有欠费红色高亮）
├── 账单分类Tab
│   ├── 全部账单
│   ├── 房屋费用（物业费、公摊水费、公摊电费等）
│   └── 停车费用（停车费、车位管理费等）
├── 账单列表（按时间倒序）
│   ├── 账单卡片
│   │   ├── 费用类型标签（物业费/公摊水费/公摊电费/停车费/车位管理费）
│   │   ├── 费用周期（如：2026年5月）
│   │   ├── 金额
│   │   ├── 状态标签（待缴费/已缴清/已逾期）
│   │   └── 缴费截止日期
│   │   └── 操作按钮（去缴费/查看详情）
│   └── 加载更多（分页）
├── 缴费记录（历史）
│   └── 已缴费列表（含支付方式、支付时间、发票状态）
└── 电子发票
    ├── 已开发票列表
    └── 申请开票（选择已缴费账单）
```

### 1.3 交互流程

```
业主进入「我的账单」
    │
    ├── 加载账单数据（调用 feeService.getBills）
    │   ├── 成功 → 展示账单总览 + 列表
    │   └── 失败 → 展示空状态 + 重试按钮
    │
    ├── 点击「去缴费」
    │   ├── 进入缴费确认页
    │   ├── 展示账单明细 + 金额
    │   ├── 选择支付方式（微信支付/支付宝）
    │   └── 确认支付 → 调起支付 SDK
    │       ├── 成功 → 跳转支付成功页
    │       └── 失败 → 提示重试
    │
    ├── 切换分类Tab
│   └── 按类别过滤（房屋费用/停车费用）展示对应账单
    │
    └── 点击「查看详情」
        └── 展示账单完整明细（费用项、计算方式、周期等）
```

### 1.4 数据字段

```typescript
type BillCategory = 'house' | 'parking';  // 房屋费用 / 停车费用
type BillType = 'property' | 'water' | 'electric' | 'parking' | 'parking_management';

interface BillItem {
  id: string;
  houseId: string;
  category: BillCategory;   // 所属类别：房屋费用 / 停车费用
  billType: BillType;
  billTypeName: string;
  period: string;           // 费用周期，如 "2026-05"
  amount: number;           // 应缴金额
  paidAmount: number;       // 已缴金额

  dueDate: string;          // 缴费截止日期
  paidAt?: string;          // 缴费时间

  itemName: string;         // 费用项名称，如"物业管理费"

  amount: number;           // 金额
}

interface BillOverview {
  totalPending: number;     // 总待缴金额
  housePending: number;     // 房屋费用待缴金额
  parkingPending: number;   // 停车费用待缴金额
  monthlyAmount: number;    // 本月应缴
  overdueCount: number;     // 欠费笔数
}

interface PaymentRecord {
  id: string;
  billId: string;
  category: BillCategory;
  amount: number;
  payMethod: string;
  tradeNo: string;
  paidAt: string;
  invoiceStatus: string;
}

interface InvoiceRecord {
  id: string;
  billId: string;
  category: BillCategory;
  billTypeName: string;
  period: string;
  amount: number;

  status: 'applied' | 'issued';
  applyTime: string;
}
```

### 1.5 校验规则

| 规则 | 说明 |
|------|------|
| 金额校验 | 支付金额必须等于应缴金额，不允许部分支付 |
| 逾期标识 | 超过截止日期未缴费自动标记为逾期，逾期账单加收滞纳金（按日计算） |
| 重复支付 | 同一账单不可重复支付，支付前检查状态 |
| 支付超时 | 支付会话有效期 30 分钟，超时自动取消 |
| 公摊水电计算 | 水电费仅计算公摊部分（公共区域分摊），屋内水电费不计入系统账单 |
| 费用项配置 | 具体费用项取决于后台费用项配置及房屋/车位创建时绑定的费用规则 |

### 1.6 非功能需求

| 项目 | 要求 |
|------|------|
| 响应速度 | 账单列表加载 ≤ 2秒 |
| 支付安全 | 接入微信/支付宝 SDK，敏感信息不经过前端 |
| 数据一致性 | 支付成功后实时更新账单状态 |
| 离线能力 | 已加载的账单数据支持本地缓存查看 |

## 二、业主端H5 - 停车缴费

### 2.1 需求概述

为业主提供车位管理、停车费缴纳、车辆出入记录查看、车位租赁、车位共享功能。支持绑定自有/租赁车位，查看车位状态和到期提醒，在线缴纳停车管理费和租赁费，查看车辆出入记录，在线租赁车位，以及将空闲时段的车位共享给他人使用并获取收益。

```
│   └── 欠费笔数提醒

│   │   ├── 车位号 + 位置描述
```

```typescript
type ParkingBindType = 'owned' | 'rented';  // 自有 / 租赁
```

### 2.5 校验规则

| 规则 | 说明 |
|------|------|
| 车位绑定 | 一户最多绑定 2 个车位 |
| 租赁冲突 | 同一车位不可同时被多人租赁 |
| 共享限制 | 仅自有车位可共享，租赁车位不可共享 |
| 费用计算 | 临时停车按小时计费，月租按月计费 |
| 租赁期限 | 最短 1 个月，最长 24 个月 |
| 共享价格 | 日间价格和夜间价格分别设置，不可为空 |
| 收益结算 | 共享收益按月结算，可提现至业主账户 |

## 三、业主端H5 - 智能门禁

### 3.1 需求概述

为业主提供手机开门功能（蓝牙/二维码/远程），支持访客授权（生成临时密码/远程为访客开门）和开门记录查询。访客授权生成的临时密码与物业管理端访客管理模块联动，访客信息自动同步至访客台账。

### 3.2 页面结构

```
智能门禁（DoorAccess.tsx）
├── 快捷开门（首页焦点）
│   ├── 默认门禁（小区大门）
│   ├── 单元门禁
│   └── 切换门禁下拉（支持搜索）
├── 开门方式（大按钮卡片式布局）
│   ├── 蓝牙开门（靠近自动感应）
│   │   ├── 蓝牙状态指示（已连接/未连接）
│   │   ├── 点击开门按钮（蓝牙已连接时可用）
│   │   └── 提示引导（未连接时引导打开蓝牙）
│   ├── 二维码开门（生成动态二维码）
│   │   ├── 二维码展示区域（居中大图）
│   │   ├── 倒计时刷新提示（30秒）
│   │   ├── 手动刷新按钮
│   │   └── 使用说明提示
│   └── 远程开门（点击按钮）
│       ├── 大按钮（点击发送开门请求）
│       ├── 加载状态（请求中）
│       └── 频率限制提示（10秒内不可重复点击）
├── 访客授权
│   ├── 生成临时密码
│   │   ├── 选择授权门禁（下拉选择）
```

## 十一、物业管理端 - 房屋/车位费用项关联管理

### 11.1 需求概述

在收费管理模块中，房屋/车位需要预先与费用项进行关联配置，才能参与账单生成。具体来说：

- **房屋费用管理：** 每个房屋可以关联多个费用项（如物业费、公摊水费、公摊电费等）。只有关联了费用项的房屋，在手动或自动生成账单时才会生成对应费用项的账单。
- **车位费用管理：** 每个固定车位可以关联多个费用项（如车位管理费）。只有关联了费用项的车位，在生成账单时才会生成对应费用项的账单。
- **费用项过滤规则：** 房屋费用管理新增费用时，费用项下拉列表仅显示收费标准中 `houseType !== 'parking'` 的费用项（排除车位类型的费用项）；车位费用管理新增费用时，费用项下拉列表仅显示收费标准中 `houseType === 'parking'` 的费用项（仅车位类型的费用项）。
- **导入功能：** 支持通过 Excel/CSV 批量导入房屋/车位与费用项的关联关系，导入时需校验房屋/车位是否存在、费用项是否存在、以及是否已存在重复关联。

### 11.5 校验规则

| 规则 | 说明 |
|------|------|
| 费用项过滤（房屋） | 房屋费用管理新增费用时，费用项下拉仅显示 `houseType !== 'parking'` 的费用项 |
| 费用项过滤（车位） | 车位费用管理新增费用时，费用项下拉仅显示 `houseType === 'parking'` 的费用项 |
| 唯一性约束 | 同一房屋/车位不可重复关联同一费用项（唯一索引：house_id + fee_item_id / parking_id + fee_item_id） |
| 导入校验-房屋存在 | 导入时校验房屋ID/名称在系统中存在 |
| 导入校验-车位存在 | 导入时校验车位ID/编号在系统中存在 |
| 导入校验-费用项存在 | 导入时校验费用项ID/名称在系统中存在 |
| 导入校验-重复关联 | 导入时校验是否已存在相同房屋/车位与费用项的关联 |
| 账单生成校验 | 自动/手动生成账单时，先查询 HouseFeeItem 关联表，仅对已关联的费用项生成账单 |
| 自定义单价 | 如填写自定义单价，则账单生成时使用自定义单价而非收费标准中的单价 |
